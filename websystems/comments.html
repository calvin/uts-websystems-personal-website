Colour Scheme

Quick note, I'm not going to go through the wireframing or the layout here because you can see for yourself how each page is laid out on this site by navigating to the 4 pages (including this one).

Colours

I started following a monochromatic colour scheme based on the base dark-purple colour, I deviated slightly with different types of purples because I thought they looked better and had better contrast on black / white.

I used less so I could use variables to store the colour values before compiling to css using the command "lessc websystems.less websystems.css"

Colours are more of a personal taste, (as long as you don't pick objectively bad colours), and I chose the dark-purple purely because it's my favorite colour and I wanted to limit myself to purples for this assessment.

In addition to the colours picked, because of the shades I picked the site should be usable / accessible to any person with any form of colour blindness, there may be issues however on extremely low contrast screens with some elements.

Colour Values

  • @dark-purple: #5e315c;
  • @purple: #753f73;
  • @light-purple: #885786;

  • @dark-grey: #3d3a3a;
  • @grey: #bababa;
  • @light-grey: #eaeaea;
  • @alternate-text-colour: white;

Layout

Each page has been built to run on IE 10+, Edge, Chrome, Mozila Firefox, Safari, and Opera.
With regards to < IE 10, users will not see the divs that are side by side in a wrapper div with display as flex.

Every Website shares the same Head, div.footer, and div.nav-container (active class set on the current page). Unique content per page is in the #main-body div.

Further to above, normally I would not repeat the navbar, footer, or head elements per page as this violates the DRY (don't repeat yourself) principle, here we have because the assessmentlimited students to use only html, css, and optionally JavaScript (but the page had to work with it disabled if needed).

The home page (index.html) is comprised of a Hero-container Div, an about-container div, and an article div.
There is an about-container with two divs side by side with content to maximize content on the page without the user having to scroll far.

Hero Classes

The hero classes relate to the Hero container on the home page, you know that big purple block with my picture in the center.

If you have used bootstrap before it looks kind of like a jumbotron, and that's what I was going for here.

.hero-container

The hero container is a parent container for which content is stored.

Here we set the background-color to the value of light-purple, center align the text (this aligns all child divs via inheritance), set the color to white (changes the tha text to #fff so it is readable and is high contrast to the light-purple background).

We also hide the overflow so that if content overflows the container we hide it, and add padding to the bottom (10% of view) and 0 the margin.

.hero-container {
    background-color: @light-purple;
    text-align: center;
    color: white;
    overflow: hidden;
    padding-bottom: 10%;
    margin-bottom: 0;
}

.hero

This class is used to deleniate the content in the hero-container, all we are doing here is adding a 75px padding to the top to give a nice gap from xl to xs screens, so you can see me in all my glory or whatever.

.hero {
    padding-top: 75px; /*so that the avatar has a nice gap between the nav and itself*/
}

.hero img

This sets the style properties for ALL IMG elements in an element with the class .hero.

I set the border-radius to 50% so that the image becomes circular, I also added -webkit and -moz for cross browser compatability. If you want to learn more you can over at W3 Schools, this is something I already knew how to do though.

In addition to making the image circular, I set the width to 200px, and using margin and border I create a solid border that is a darker purple than the background light-purple for well, because I thought it looked nice.

.hero img{
    border-radius: 50%;
    -webkit-border-radius: 50%;
    -moz-border-radius: 50%;
    margin: 4px;
    border: 4px solid @purple;
    width: 200px;
}

Read the comment, in the example div. This is a work around for making child divs in a wrapper div float next to each other nicely, like these divs or the about / contact div on the home page.

Media Classes

Here I'll break down the @media css bit.

@media screen and (max-width: 1000px)

This says that should the screen width go below 1000px then do the following:

  • Set display to block for abour-container
  • set width to 90% for classes .about, .contact, .work-experience, #work-experience, .academic-experience, #academic-experience, and .example
  • Hides all anchor tags that are in a div that has the class .navbar-collapsable, and then overrides this by showing any links in navbar-collapsable, that are also in a div with the class dropdown.

Let's break it down.

about-container goes to block, it was felx before so we could fit the divs side by side and make them the same height as the div to the left. This needs to be set to block so that we can make the divs "fold underneath" one-another for responsive design on smaller displays

all the classes for divs that are in an about-container have there width set to 90%, (5% either side for padding + content at 90% = 100% screen width), so that as they are floating left, they will "fold underneath" eachother (right hand side div folds under left hand side).

.navbar-collapsable is used only on the websystems part of the assessment, not in the main page. this is used to make navigating the website on a phone easier on a mobile.

@media screen and (max-width: 1000px) { /*index break about and contact and make width 100%*/
    .about-container {
        display: block
    }

    .about, .contact, .work-experience, #work-experience, .academic-history, #academic-history, .example {
        width: 90%;
    }

    .navbar-collapsable li a {
        display: none;
    }

    .navbar-collapsable li div.dropdown, .navbar-collapsable li div.dropdown a{
        display: block;
    }
}

JavaScript Rundown

This site has a very limited amount of JavaScript, mainly for a scroll to top function, showing and hiding these accordions, and smooth scrolling.

Please Note: if you have view issues with the code in the example div, you can also inspect the page and view the scripts under source > websystems.js

toggleAccordian(id)

This function toggles an accordian div from show to hide and vice versa.

First we get the element by it's id, then if the class name doesn't have show-accordian we replace hide-accordian with show-accordian and vice versa.

// Accordian, from https://www.w3schools.com/w3css/tryit.asp?filename=tryw3css_accordion 
// addapted so I do not have to use !Importa
function toggleAccordian(id) {
    var x = document.getElementById(id);
    if (x.className.indexOf("show-accordian") == -1)
        x.className = x.className.replace("hide-accordian", "show-accordian");
    
    else 
        x.className = x.className.replace("show-accordian", "hide-accordian");
}

window.onload

on window.onload execute a function that goes through and sets accordiands to be closed. They are open by default so that if JavaScript is disabled, the user can still view the content.

// if accordions, toggle close (they are open by default incase JavaScript is disabled).
// see this excellent answer for why I wish I could use JQuery $(document).ready would have sufficed as I just need to maniplate the accordions when the DOM is ready.
window.onload = function() {
    var accordions = document.getElementsByClassName("show-accordian");

    // we need to get the ids in an array then toggle each, we do this because if we change the class we are selecting on then the array accordions will be shortened each loop, 
    // try it out if you don't believe me (god I hate vanilla JavaScript, it's like a throw back to the hell of yesteryear).

    var accordianIds = []
    var accordionsLength = accordions.length;

    for (var i = 0; i < accordionsLength; i++)
        accordianIds.push(accordions[i].id);

    for(var i = 0; i < accordionsLength; i ++)
        toggleAccordian(accordianIds[i]);
}
